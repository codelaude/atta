# Pattern Files

> This directory contains technology-specific pattern files auto-generated by the `/atta` skill.

## How It Works

Run `/atta` in your project to automatically detect your tech stack and generate pattern files here. For example:

- If Vue is detected: `vue-patterns.md` is created
- If React is detected: `react-patterns.md` is created
- If SCSS is detected: `scss-patterns.md` is created
- If TypeScript is detected: `typescript-patterns.md` is created
- If Jest/Vitest is detected: `testing-patterns.md` is created

## Manual Creation

You can also create pattern files manually following this structure:

```markdown
# [Technology] Patterns

## Key Rules
- [Rule 1]
- [Rule 2]

## Anti-Patterns to Flag
| I See | I Do | Severity |
|-------|------|----------|
| [pattern] | [fix] | CRITICAL/HIGH/MEDIUM |

## Examples
[Code examples specific to your project]
```

The Librarian agent (`/atta-agent librarian`) can also propose updates to pattern files based on learnings captured during development.
