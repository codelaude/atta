# Legacy Agents (v1.0)

These agents are from the original Vue/TypeScript/SCSS-focused implementation.

## Purpose of This Directory

In **v2.0**, tech-specific agents are **dynamically generated** from templates based on your project's detected stack. The agents in this directory are preserved as:
- **Reference material** for understanding the original system design
- **Examples** of what fully-populated agents look like
- **Historical record** in case you need to reference the original rules

## Do Not Use These Directly

These agents will **not** be loaded or invoked by the system. They exist only for reference.

## Current Agents

### Coordinators (replaced by generated coordinators in v2.0)
- **fe-team-lead.md** - Frontend coordinator (v1.0 static version, now generated by `/atta`)
- **backend-consultant.md** - Backend advisor (v1.0 advisory-only mode, replaced by `be-team-lead` in v2.0)

### Specialists (replaced by generated specialists in v2.0)
- **vue.md** - Vue.js specialist (Composition API, reactivity, component structure)
- **scss.md** - SCSS/BEM specialist (styling, theming, responsive design)
- **typescript.md** - TypeScript specialist (type safety, interfaces, generics)
- **tester.md** - Testing specialist (Jest/Vitest, Vue Test Utils)
- **accessibility.md** - WCAG/ARIA specialist (now in templates, still a good reference)

## What Replaced These?

In v2.0:
- **Templates** in `.claude/bootstrap/templates/agents/` define universal agent structures
- **Detection rules** in `.claude/bootstrap/detection/` identify your tech stack
- **Mappings** in `.claude/bootstrap/mappings/agent-mappings.yaml` connect detected tech to templates
- **Generation** happens during `/atta`, creating specialists in `.claude/agents/specialists/`

## Migration Note

If you ran `/migrate` from v1.0 to v2.0, any custom rules you added to these agents were preserved and merged into the new generated agents.
