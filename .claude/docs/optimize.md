# Prompt Optimizer

**Status**: Core feature (v2.7)

## Overview

The prompt optimizer enriches your prompts with project context for better results. It has two main use cases:

1. **Same-session optimization** — Rephrase or restructure a prompt that didn't get a good result, or approach a problem from a different angle with better context injection.
2. **Cross-tool handoff** — Inject project context into a prompt so another AI tool (Codex, Copilot, ChatGPT, Gemini) can work effectively without access to your project.

## Quick Start

```
/optimize Add a dark mode toggle to the settings page
/optimize --rephrase How should I structure the auth module
/optimize --target codex Refactor the auth module to use JWT
/optimize --target chatgpt Explain how our routing works and suggest improvements
```

## How It Works

1. **Reads project context** — tech stack, paths, build commands, architectural patterns from `project-context.md`
2. **Selects relevant patterns** — only the framework/language rules that apply to your task
3. **Adapts to the mode** — same-session prompts stay conversational; cross-tool prompts are formatted for the target
4. **Outputs a copyable prompt** — ready to use in this session or paste into another tool

## Modes

| Mode | Flag | What It Does |
|------|------|-------------|
| Same-session | (default) | Enriches prompt with context for use in this conversation |
| Rephrase | `--rephrase` | Restructures a prompt that didn't get good results — tries a different angle |
| Codex | `--target codex` | Explicit file instructions, leverages repo access |
| Copilot | `--target copilot` | Concise — focuses on what Copilot can't infer |
| ChatGPT | `--target chatgpt` | Self-contained with full context |
| Gemini | `--target gemini` | Structured sections, can include more detail |
| Claude | `--target claude` | Summary + pattern references |

## What Gets Injected

The optimizer reads from `project-context.md` (generated by `/atta`) and selectively injects:

- **Tech stack** — always included (framework, language, versions)
- **Architectural patterns** — when the task involves structure or organization
- **Conventions** — when the task involves writing code
- **Build commands** — when the task involves running or testing
- **Preferences** — when available from developer profile

It does **not** inject Atta internals, session data, secrets, or full pattern file dumps. Context is distilled to what matters for the specific task.

## Prerequisites

Run `/atta` first to generate `project-context.md`. Without it, `/optimize` has no context to inject.

For better results, also run `/profile --update` to set your preferences — they'll be included in enriched prompts.

## Examples

### Rephrasing in the current session

```
/optimize --rephrase How should I structure the auth module
```

If your first attempt at asking this question got a generic answer, `/optimize` restructures the prompt with your project's architectural patterns, tech stack, and conventions — so the next response is grounded in your actual project.

### Enriching for Codex

```
/optimize --target codex Add pagination to the user list API endpoint
```

Produces a prompt with file paths, API conventions from your project, and explicit "modify file X" instructions that Codex works best with.

### Enriching for ChatGPT

```
/optimize --target chatgpt Review our authentication flow and suggest improvements
```

Produces a self-contained prompt with full tech stack, auth-related patterns, and architectural context — since ChatGPT has no access to your repo.
