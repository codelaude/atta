# Quality Gate

Universal checks covering the full implementation cycle — from investigation through completion. For full validation with project-specific pattern checks, use `/atta-preflight`.

## Before Making Changes

- [ ] **Read first** — Read target files fully before modifying. Understand current behavior.
- [ ] **Research** — Verify approach against external sources (official docs, web search). Don't rely solely on model knowledge for API behavior, shell semantics, or security claims.
- [ ] **Change once** — Make the change correctly the first time. Avoid iterative guess-and-fix loops.

## During Implementation

- [ ] **New modules** — Write tests alongside implementation, not after.
- [ ] **Hardcoded values** — Verify URLs, paths, version numbers match their source of truth in the project.
- [ ] **Unused code** — Check for dead imports and unreferenced variables before declaring done.

## Before Declaring Done

### Must Pass

- [ ] **Tests pass** — Run the project's test suite. Don't skip failing tests.
- [ ] **Lint clean** — No violations against the project's configured linter rules.
- [ ] **Security clear** — No hardcoded secrets, no code injection vectors (eval/exec with user input, unsanitized HTML rendering, SQL string concatenation).
- [ ] **Cross-file consistency** — If you changed a shared value (URL, version, config key, API endpoint), verify all other references match.

### Should Verify

- [ ] **Platform portability** — No hard-coded OS assumptions in shipped code (path separators, shell commands, line endings).
- [ ] **Acceptance criteria** — If ACCs exist for this task, verify each criterion is met.
- [ ] **Project patterns** — Check `.atta/team/patterns/` for project-specific conventions (generated by `/atta`). Review against those if present.

## Completion Workflow

```
1. /atta-preflight    — Static analysis + lint + security + tests + review (project-aware)
2. Fix any issues
3. /atta-ship         — Tests + ACCs + PR description + learnings capture
```

### PR Description

- Follow template in `.atta/team/templates/pr-template.md` if present
- Include: Summary (1-3 bullets), Changes (grouped), Verification (checklist), Notes (breaking changes)
- PR title under 70 characters

### Learnings Capture

Before closing a session with significant work:
- Were there corrections or surprises? Capture via `/atta-librarian`
- Did an assumption prove wrong? Record why
